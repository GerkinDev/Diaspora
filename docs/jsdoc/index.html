Modules architecture

For now, modules are divided in 3 categories:

  1. Extensions
  2. Functionnal modules
  3. Terminal modules

An application (like a full website) is a set of Terminal modules that defines each a feature.

Extensions

Extension modules define convinient ways to extend every dependant modules. An extension links a module's key with its handling function with the extendModule method of AltCore.

They can use both Extensions and Functionnal modules, and are meant to define generic logic

Functionnal modules

Functionnal modules bring new features to its children by defining APIs. Functionnal modules are almost the same as extension, except that they do not use extendModule.

They can use both Extensions and Functionnal modules, and are meant to define generic logic

Terminal modules

Terminal modules are the only types of modules that can contain specific logic. When you want to start an app feature, you usually do a loadModule with the appropriate Terminal Module that will include its dependencies.

They can use Extensions, Functionnal modules and other Terminal modules.
